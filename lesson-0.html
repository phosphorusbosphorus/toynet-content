A Motivating Example

example: Host-to-Host Communication (When There is No Network)

Intro

  • You ask for a copy of a Powerpoint file on my computer (that I am making for a joint training brief!)

Setup

  • I use Computer Alpha.
  • You use Computer Bravo.
  • Computer Alpha and Computer Bravo may be two of many computers in the vicinity.
  • Computer Alpha and Computer Bravo have no network connecting them.

Goal

  • I want to send data from Computer Alpha to Computer Bravo.

Steps

  • I plug in a USB to Computer Alpha.
  • I copy the data from Computer Alpha to the USB.
  • I walk over to Computer Bravo.
  • I plug in the USB to Computer Bravo.
  • I copy the data from the USB to Computer Bravo.

Room for Improvement

  • How can we set up a network, so that this operation can be done without a USB, without physically walking over?

The OSI 7-Layer Network Model

A Model

Definition

  • A model is a simplified representation of a real thing.

A Network Model

Definition

  • A network model is a simplified representation of a network,
    so that a technician can conceptualize the network when troubleshooting problems,
    so that technicians can easily communicate with each other, and
    so that different manufacturers can design devices to be compatible.

The OSI 7-Layer Network Model

Definition

  • The Open Systems Interconnection 7-Layer Network Model or OSI 7-Layer Network Model conceptualizes a network as 7 layers:
    • Layer 7 or Application Layer
    • Layer 6 or Presentation Layer
    • Layer 5 or Session Layer
    • Layer 4 or Transport Layer
    • Layer 3 or Network Layer
    • Layer 2 or Data Link Layer
    • Layer 1 or Physical Layer
  • (We will observe each layer in each lesson module.)

The TCP/IP Network Model

Definition

  • The Transmission Control Protocol/Internet Protocol Network Model or TCP/IP Network Model conceptualizes a network as 4 layers:
    • Application Layer
    • Transport Layer
    • Internet Layer
    • Link Layer or Network Interface Layer

example: No Network

  • There are several computers in the vicinity.
  • How can we add devices here to form a network, through which Computer Alpha can send data to Computer Bravo?

example: OSI Layer 1: Physical

  • We can add cables and hubs to connect all the computers together in some formation.
  • A hub is a device that receives data, then sends a copy of the data to each connected computer except for the source computer.
  • (We'll later see more of cables and hubs, which are concepts in OSI layer 1.)

example: OSI Layer 2: Data Link

  • Instead of a hub, we can connect a switch.
  • A switch is a device that receives data, then sends the data to just the destination computer.
  • (We'll later see more of switches, which are concepts in OSI layer 2.)

example: OSI Layer 3: Network

  • In order to connect even more computers, we can connect more switches and routers.
  • A router is a device that receives data, then sends the data to another device that is closer to the destination computer (or to the destination computer directly, if possible).
  • (We'll later see more of routers, which are concepts in OSI layer 3.)
  • In this setup, Computer Alpha and Computer Bravo may share a switch, ...

  • ...or Computer Alpha and Computer Bravo may not share a switch but still share a router, ...

  • ...or Computer Alpha and Computer Bravo may not even share a router.
  • (We'll later see how Computer Alpha sends data to Computer Bravo in each case.)
